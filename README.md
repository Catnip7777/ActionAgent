# LLM Action Agent 四个版本整体介绍

> 从 v2.2 全功能版到 v3.10 轻量版——两条升级路径，四个关键版本，一个核心使命：让大模型直接操作你的本地文件。

---

## 一、版本家族总览

### 两条独立路线

```
全功能版： v2.2 —————→ v3.0      （功能完整，可选后端）
                  ↘
轻量版：            v3.6 ————→ v3.10  （纯浏览器，最小可用）
```

### 四个版本速览

| 版本 | 定位 | 面板数 | 后端依赖 | 核心特色 |
|------|------|:------:|:--------:|---------|
| **v2.2** | 全功能版起点 | 10 | server.py | 插件/记忆/统计/日志全家桶 |
| **v3.0** | 全功能版升级 | 9 | server.py | 🆕 文件上下文池 + 多工作区 + Token 统计 |
| **v3.6** | 轻量版起点 | 3 | 有后端配置 | 3面板极简 + 编码原则 + 任务板先行 |
| **v3.10** | 轻量版最终版 | 3 | ❌ 纯浏览器 | 🆕 文件上下文池 + stale 标记 + 零配置 |

### 一句话描述

> **v2.2** 是一切开始的地方 → **v3.0** 引入文件上下文池改变交互方式 → **v3.6** 剥离出纯浏览器路线 → **v3.10** 完善上下文池并彻底去后端。

## 二、定位演变

### 全功能版路线（v2.2 → v3.0）

**目标用户**：需要完整功能套件的开发者
- 插件系统（可复用的工作流）
- 持久记忆（让 AI 记住长期信息）
- 统计面板（Chart.js Token 趋势图 + 操作分类）
- 执行日志（完整操作记录）
- 后端 server.py（突破浏览器文件系统限制）
- 多工作区手柄（多个目录并行操作）

**用户画像**：
- 需要 AI 处理多个项目的开发者
- 对 Token 消耗敏感的重度用户
- 需要使用后端操作本地任意文件的场景

### 轻量版路线（v3.6 → v3.10）

**目标用户**：只需要核心对话 + 文件操作 + 任务板的普通用户
- 3 个面板：对话 / API 设置 / 工作区
- 无插件、无记忆、无统计、无日志面板
- 从「有后端配置」简化到「纯浏览器，即开即用」
- 纯前端数据，数据不出浏览器

**用户画像**：
- 双击 index.html 就开用的懒人
- 日常辅助编程、做文件批量修改
- 不需要搭服务器的任何人

## 五、升级脉络与关键里程碑

### v2.2 → v3.0：全功能版升级

**里程碑**：引入文件上下文池

主要变化：
1. **对话对象新增** `contextFiles` 和 `stats` 字段
2. **新增 API**：addContextFile / removeContextFile / getContextFiles / onContextFileWritten / getFileContextSection
3. **提示词从三段式变为动态段**：`getSystemPromptCore() + getDynamicEnvSection()`
4. **新增多工作区字典**：workspaceHandles 取代单一 handle
5. **新增对话级 Token 统计**：包括缓存命中率
6. **新增 action-editfile.js**：7种安全编辑操作
7. **移除日志面板导航**：面板数 10→9
8. **代码量精简**：app-init.js 减少 43%（381→216行）

### v3.6 → v3.10：轻量版升级

**里程碑**：引入文件上下文池 + 完成纯浏览器化

主要变化：
1. **对话对象新增** `contextFiles` 字段（v3.6 没有）
2. **新增 API**：addContextFile / removeContextFile / getContextFiles / onContextFileWritten / getFileContextSection / clearContextFiles
3. **独有特性：stale 标记** `___STALE___`（全功能版不具备）
4. **提示词从三段式变为四段式**：`buildSystemPromptCore() + workspace + tasks + fileContext`
5. **移除后端依赖**：settings 面板中的后端地址/Token/测试连接全部移除
6. **移除附加工具栏**：+动作/+插件/+记忆/+文件/+任务 全部移除
7. **移除手动动作面板**
8. **对话统计升级**：新增 getConversationStats 函数
9. **编码原则更新**：read_file → add_file_to_context
10. **代码增加**：app-core.js 17.2KB→21.8KB（+4.6KB）

## 八、总结

### 两条路线的共同进化方向

1. **文件上下文池成为核心**：v2.2/v3.6 没有 → v3.0/v3.10 成为标配
2. **交互方式改变**：从「一次性读取全部代码」到「按需读取关键行」
3. **去重简化**：全功能版移日志面板，轻量版移除后端和附加面板
4. **体验提升**：自动更新、行号引用、过期提示

### 两条路线的各自特色

- **全功能版**：功能完整但配置门槛高，适合重度用户和团队
- **轻量版**：即开即用但功能有限，适合日常使用的个人用户
- **v3.0** 独有的多工作区和统计系统
- **v3.10** 独有的 stale 标记和完善的编码原则

### 终极选择

> **日常用 v3.10，重度用 v3.0。**
